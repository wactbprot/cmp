build-mpd

(build-mpd)(build-mpd mp-id)

Loads mpd from long term memory and builds the short term memory. The mp-id must be set with workon!.

Usage:

(build-mpd mpid)
;; or
(workon! mpid)
;; followed by
(build-mpd)
(check)
(start)

(reset-c i)

Shortcut to push a reset to the control interface of mp container i. The reset cmd does not de-register the state listener so that the container starts from the beginning. reset is a container restart

set-ctrl

(set-ctrl i cmd)(set-ctrl mp-id i cmd)

Push a command string (cmd) to the control interface of a mp. cmds are:

  • "run"
  • "stop"
  • "mon"
  • "suspend"

The mp-id is received over (->mp-id). NOTE: The definitions struct should not be started by user (see workon!).
