cmp.work

dispatch

(dispatch x)

Dispatches to the workers depending on (:Action task).

Since every worker has to manage the state, the state-key is the second parameter.

(dispatch {:Action "wait" :WaitTime 1000 :StateKey "testpath"})
(dispatch! {:Action "foo" :StateKey "testpath"})
;; ERROR [cmp.work:52] - unknown action:  :foo

get-task

(get-task x)

Returns the assembled task for the given key k related to the proto-task. Since the functions in the cmp.task namespace are (kept) independent from the tasks position, this info (:StateKey holds the position of the task) have to be assoced in tsk/assemble.
