225  (defn set-ctrl
226    "Writes the command string (`cmd`) to the control interface of a
227    `mpd`. If the `mpd` is already started (see [[m-start]]) the next
228    steps work as follows: `cmd` is written to the short term memory by
229    means of [[cmp.st-mem.set-val!]].  The writing process triggers the
230    `registered` `callback` (registered by [[m-start]]). The `callback`
231    cares about the `cmd`.  `cmd`s are:
232  
233    * `\"run\"`
234    * `\"stop\"`
235    * `\"mon\"`
236    * `\"suspend\"`
237  
238    The `mp-id` parameter may be give directly
239  
240    ```clojure
241    (set-ctrl \"ref\" \"run\")
242    ```
243    or is derived from `@current-mp` [[workon \"ref\"]].
244  
245  
246    ```clojure
247    (workon! \"ref\"=
248    (set-ctrl \"run\")
249    ```
250  
251    **NOTE:**
252  
253    `set-ctrl` only writes to the `container` structure.  The
254    `definitions` struct should not be started by a
255    user (see [[workon!]])."
256    ([i cmd]
257     (set-ctrl @current-mp i cmd))
258    ([mp-id i cmd]
259     (st/set-val! (ku/cont-ctrl-key mp-id (u/lp i)) cmd)))
